### 1. **用「角色鏈」提升 Claude 長文創作品質**

**帳號：** @danshipper（Dan Shipper，Every.to 創辦人）

**類型：** 📝 提示技巧

**核心方法/技巧：**

- 將寫作任務拆成三個獨立角色：Researcher（收集論點）→ Structurer（建立大綱）→ Writer（撰寫文字）
- 每個角色用獨立的 prompt 呼叫，不在同一個對話中混合任務
- 最後用第四個角色 Editor 做批評性回顧，找出邏輯漏洞和語氣不一致處

**為什麼有用：**

單一大 prompt 容易讓 LLM 在多個任務間妥協品質。角色鏈讓每個步驟專注於單一目標，長文品質大幅提升，修改次數減少 60%。

**編輯觀點：** 💬 這個方法的核心是「關注點分離」，與軟體工程的 SRP 原則一樣——每個函式只做一件事，AI 寫作也應如此。

**天馬行空的延伸：** 🚀 把角色鏈存成可復用的「寫作 SOP」，針對不同內容類型（教學文、分析文、故事文）各建一條，像切換裝備一樣快速調用。

**💬 最有價值的回應：**

> 我試了這個方法寫技術文章，最大的發現是 Structurer 那一步——讓 AI 先列出「哪些論點我不應該放進來」比「哪些要放」更有用，幫我省掉了一堆贅肉。

### 3. **用 AI 做「讀者雷達」：讓內容策略數據化而非感覺化**

**類型：** 💡 工作流優化

**帳號：** @emollick（Ethan Mollick，Wharton 教授）

**核心方法/技巧：**

- 每週把你過去 20 篇文章的標題和互動數（like、share、comment）丟給 AI，請它找出「高互動文章的共同模式」
- 讓 AI 分析你的受眾最在意的三個核心問題是什麼
- 用這份分析生成下週的「內容假設清單」，每篇文章都是一個可測試的假設

**為什麼有用：**

大多數內容創作者靠直覺選題，但 AI 能在幾分鐘內從大量數據中找到你自己都沒察覺的規律，讓內容策略從「猜測」變成「實驗」。

**編輯觀點：** 💬 把創作行為「科學化」並不會失去創意，反而讓你有更多精力放在真正有創意的部分，因為選題的心智負擔被消除了。

**天馬行空的延伸：** 🚀 結合競品帳號的公開數據，讓 AI 找出「你的受眾喜歡但你沒做過的題材」，發現內容藍海而非跟風做熱門話題。

**💬 最有價值的回應：**

> 我加了一個步驟：每次分析完，讓 AI 扮演「懷疑論者」反駁這些結論——避免過度相信 pattern，因為相關性可能只是巧合。

### 4. **「Context Window 管理術」：讓 Claude 長對話不失憶的四個技巧**

**帳號：** @karpathy（Andrej Karpathy，前 OpenAI 研究員）

**類型：** 📝 提示技巧

**核心方法/技巧：**

- **Rolling Summary**：每 10 輪對話，請 AI 生成一個「截至目前的對話摘要」，下次用摘要開頭新對話
- **State File**：維護一個純文字的「專案狀態檔」，每次對話開頭貼上，結尾更新
- **Numbered Decisions**：重要決定用編號記錄（Decision #1: 用 Python 而非 JS），方便 AI 快速定位
- **Context Budget**：明確告訴 AI「你只有 1000 tokens 的背景資訊」，強迫它做重要性排序

**為什麼有用：**

長專案中 AI 失憶是最大痛點。這四個技巧讓你跟 AI 的協作跨越 context window 限制，維持連貫的工作狀態，特別適合需要多天完成的寫作或研究專案。

**編輯觀點：** 💬 把 AI 當成有記憶障礙的超級聰明助理——你需要幫它記憶，但它的即時處理能力遠超你，這個組合才是最強搭配。

**天馬行空的延伸：** 🚀 把 State File 版本化（用 Git 管理），讓你可以「回溯」到任何一個專案狀態重新開始，AI 協作也有了時間旅行能力。

**💬 最有價值的回應：**

> 我在 State File 裡加了「這個 AI 對話的目的是 X，成功標準是 Y」，讓 AI 每次都有明確評估框架，避免它把對話帶到錯誤方向。

### 5. **「AI 創作飛輪」：如何讓每一篇內容都餵養下一篇**

**帳號：** @ruanyf（阮一峰，科技週刊作者）

**類型：** 🛠️ 可複用方法

**核心方法/技巧：**

- 每篇文章發布後，讓 AI 從讀者留言中提取「未被回答的問題」，這些就是下篇文章的選題
- 用 AI 把一篇長文拆成 5 個不同格式（短推、FAQ、清單、故事、數據可視化），一次創作多次分發
- 建立「常青素材庫」：每次 AI 生成的好例子、好比喻，存進 Notion，下次創作時當作 seed prompt

**為什麼有用：**

內容創作者最大的資源浪費是「每次創作都從零開始」。飛輪方法讓每篇內容都成為下一篇的原料，創作效率隨時間複利成長，而非線性成長。

**編輯觀點：** 💬 最好的內容策略不是「多產」而是「高效複用」——一個好的核心洞見，可以在六個平台、三種格式、兩個目標受眾中生命延續三個月。

**天馬行空的延伸：** 🚀 把「素材庫」訓練成個人 RAG 系統，讓 AI 在生成新內容時自動引用你過去說過的話，建立一致的個人知識體系和寫作風格。
