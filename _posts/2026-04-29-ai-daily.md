### 1. **Cursor 3.2 /multitask：一個指令，多個 AI Agent 同時非同步工作**

**帳號：** @dotey（宝玉）/ Cursor 官方

**類型：** 💡 工作流優化

**核心方法/技巧：**

- 新指令 `/multitask`：讓 Cursor 把你的請求拆成小塊，交給多個**非同步子 Agent 同時執行**，而不是排隊等候
- **Worktrees（工作樹）**：在不同 branch 上跑隔離任務，完成後一鍵合併到本地工作區
- **Multi-root Workspaces（多根工作區）**：一個 Agent session 同時跨越 frontend、backend、shared library 三個 repo，不用反覆切換
- 舊做法：等 → 等 → 等（循序排隊）；新做法：全部同時跑，等最慢那個

**為什麼有用：**

對需要同時處理多個內容模塊的創作者而言，/multitask 讓你「一個想法，多個 Agent 同時展開」——草稿、SEO 分析、社群摘要三件事不再是三輪等待，而是一輪並行。

**編輯觀點：** 💬 Cursor 從「更好的編輯器」進化到「多 Agent 協調平台」的速度比大多數人預期的快——這次更新讓 Cursor 和 Claude Code 的功能邊界越來越模糊，競爭很快會變成誰的 Agent 協調更聰明。

**天馬行空的延伸：** 🚀 把 /multitask 套在內容生產流水線上：一個 Agent 寫長文、一個拆成 Twitter thread、一個做 LinkedIn 貼文版本、一個生成 email newsletter 版——四種格式，一次並行，最後你只要選哪個最好。

**推文連結：** [https://x.com/dotey/status/2047790274949132357](https://x.com/dotey/status/2047790274949132357)

### 2. **X Custom Timelines：用 Grok 訂閱 75+ 話題，把 X 變成你的專屬新聞台**

**帳號：** @nikitabier（Nikita Bier，X 產品負責人）

**類型：** 🚀 新工具

**核心方法/技巧：**

- **Custom Timelines**：在主頁 Tab 釘選特定話題，支援 75+ 話題分類（AI、科技、創業、設計……）
- 由 Grok 驅動，根據你在該話題的互動行為持續個人化——越用越精準
- 額外功能：「**Snooze Topic**」——在 For You 頁暫停你不想看的話題（對創作者：暫停熱門但干擾的話題，保持創作專注）
- 現在可用：iOS + Web，Premium 訂閱者優先開放

**為什麼有用：**

內容創作者最大的挑戰之一是「持續找到值得追的 AI 動態而不被雜訊淹沒」——Custom Timelines 讓你不需要手動策展 list，Grok 自動幫你做話題過濾和個人化。

**編輯觀點：** 💬 這是 X 少數幾個真正讓我想更常用 X 的功能——把平台從「看演算法給什麼」變成「我決定我看什麼」，方向完全正確；唯一擔心的是 Grok 的話題理解是否夠準確，特別是中文語境。

**天馬行空的延伸：** 🚀 把 Custom Timelines 當成「競品雷達」：訂閱你最強的 5 個競爭對手的同類話題，每天早上掃一眼他們在追什麼新話題、新工具——比任何競品分析報告都更即時。

**推文連結：** [https://x.com/nikitabier/status/2046736181002645520](https://x.com/nikitabier/status/2046736181002645520)

### 3. **Google Workspace Studio 全面開放：用說話就能在 Google Workspace 建 AI Agent**

**帳號：** @GoogleAI / @AnthropicAI

**類型：** 🚀 新工具

**核心方法/技巧：**

- **Workspace Studio**（workspaceupdates.googleblog.com）：無需寫程式，用自然語言描述想自動化的工作，Gemini 直接生成 Agent
- 可自動化範疇：Gmail 智能分類回覆、Drive 文件整理、Sheets 資料分析、支援工單分流、內容生成工作流
- **Workspace Intelligence**：讓 Gemini 即時讀取你的 Gmail、Calendar、Chat、Drive，生成回應時自動帶入你的工作脈絡
- 現在對所有 Workspace 用戶開放（至 6/1 前有較高使用配額）

**為什麼有用：**

對重度使用 Google Workspace 的創作者而言，這是「免費的 AI 自動化秘書」——幫你分類粉絲來信、整理每週素材庫、自動更新內容日曆，不需要任何程式技能。

**編輯觀點：** 💬 Google 把 Workspace Studio 做成「描述問題 → AI 自動建流程」的模式，比 Zapier 更聰明、比 n8n 更容易上手——但目前 Google Workspace Agent 的能力上限還不如 Claude Code，適合做「規律性、重複性」的任務自動化。

**天馬行空的延伸：** 🚀 建一個「內容創作者晨間 Agent」：每天 8AM 自動掃描你訂閱的 Gmail newsletters、把昨日的 AI 重點摘要成 3 點、存進 Google Docs、再附上你昨天在 Calendar 裡標記的待辦事項——你的 AI 晨報，不需要任何訂閱費。

**推文連結：** [https://workspaceupdates.googleblog.com/2025/12/workspace-studio.html](https://workspaceupdates.googleblog.com/2025/12/workspace-studio.html)

### 4. **emollick：AI 可以學會「品味」——被引用的預測，改變你對 AI 創作的理解**

**帳號：** @emollick（Ethan Mollick）

**類型：** 📝 提示技巧

**核心方法/技巧：**

- 最新研究：一個小模型只靠「論文引用數」訓練，就能**預測哪些論文會爆紅**
- 核心邏輯：引用、按讚、分享——這些「參與度訊號」可以教 AI 判斷品質，而不只是執行
- 對創作者的啟示：用你的**高互動內容**（而非你的所有內容）微調 AI 的品味偏好，讓它學會你的讀者喜歡什麼
- 實作方法：把你按讚或收藏的前 100 篇文章餵給 AI，告訴它「學習這些文章的共同點」

**為什麼有用：**

大多數人把 AI 當執行工具，這個研究說明 AI 也可以成為你的「品質顧問」——只要你餵給它正確的訓練訊號（你的過去互動紀錄），它就能幫你判斷「這個想法/標題/文章有沒有爆的潛力」。

**編輯觀點：** 💬 「AI 能學品味」這件事讓內容創作的護城河突然變深了——懂得用高品質互動訊號訓練 AI 偏好的創作者，將擁有比別人更精準的 AI 品味過濾器，這是一個不容易被複製的個人優勢。

**天馬行空的延伸：** 🚀 把你過去三年最高點讚的 50 篇貼文和最低的 50 篇做成對比資料集，讓 AI 分析兩組的語氣、結構、開場方式有什麼系統性差異——這份「你的個人品味報告」比任何內容顧問的建議都更準確。

**推文連結：** [https://x.com/emollick/status/2035387292311769278](https://x.com/emollick/status/2035387292311769278)

### 5. **emollick：AI 讓職業邊界重劃——你的護城河在哪裡？**

**帳號：** @emollick（Ethan Mollick）

**類型：** 🛠️ 可複用方法

**核心方法/技巧：**

- AI 打破舊職業分工後，各專業將**爭奪新的職責邊界**（引用 Abbott「職業體系」理論）
- 歷史規律：每次重大技術變革，技術能做的工作會從某些職業「流出」，被鄰近職業「吸收」
- 對創作者的策略：不是問「AI 會取代我嗎」，而是問「AI 讓哪些原本不屬於我的工作，現在我可以佔領」
- 具體行動框架：列出你領域內 5 件「以前需要其他專業」的事，評估 AI 能否讓你直接做這些事

**為什麼有用：**

與其焦慮 AI 會搶走什麼，不如主動思考 AI 讓你能搶占哪些新領域——這是一個積極進攻的框架，而不是被動防守。

**編輯觀點：** 💬 Ethan 這篇的核心洞察是：AI 不只是壓縮你的護城河，它也在開放別人的護城河讓你進去——問題是你得先看見這個機會窗口，而大多數人還在用防守心態思考 AI。

**天馬行空的延伸：** 🚀 想像一個「職業邊界地圖」工具：輸入你的職業，AI 幫你分析哪些鄰近職業的工作因為 AI 而變得容易跨入——你的下一個收入來源，可能是五年前需要整個另一個學位才能做的事。

**推文連結：** [https://x.com/emollick/status/2048480177210958193](https://x.com/emollick/status/2048480177210958193)
