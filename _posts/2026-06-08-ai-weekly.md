### 1. **Loop Engineering 五要素：不再手動 Prompt，設計自動跑的 Agent 循環**

**帳號：** @addyosmani

**類型：** 🛠️ 可複用方法

**核心方法/技巧：**
- **五要素**：Automations（定時發現任務）、Worktrees（隔離並行作業）、Skills（避免每次重新說明情境）、Plugins/Connectors（連接真實工具）、Sub-agents（製作者與審查者分離）
- **第六要素：記憶**——一個存在 session 外的 Markdown 檔或 Linear Board，讓 Agent 跨次記住進度
- Boris Cherny（Claude Code 負責人）名言：「我不再 prompt Claude 了，我有循環在跑，它們替我 prompt Claude，我的工作是寫循環。」

**為什麼有用：**
把你從「每次都在打指令」的工作模式，升級成「設計一個會自動找任務、分配任務、審查任務的小工廠」，一次設定，持續運轉。

**編輯觀點：** 💬 這篇文章把 Agent Harness、Orchestration、Loop 三個概念的關係說清楚了——如果你一直搞不清楚這些術語，這篇是目前最好的入口。

**天馬行空的延伸：** 🚀 Loop Engineering 用在個人品牌會怎樣？每天早上一個 Automation 掃描你訂閱的 100 個 RSS，一個 Sub-agent 篩選，另一個 Sub-agent 草稿你的每日 Threads 貼文，你只需審核。完全靠 Skills 記住你的寫作風格。

**💬 最有價值的回應：**
> "The five pieces map almost exactly onto both Codex and Claude Code, which means once you design a loop, it works in either tool."
— 摘自 addyosmani 文章本身（@addyosmani），提醒工具中立性：設計好的 Loop 跨工具通用。

**推文連結：** https://addyosmani.com/blog/loop-engineering/

### 2. **Orchestration Tax：你才是多 Agent 系統的瓶頸**

**帳號：** @addyosmani

**類型：** 💡 工作流優化

**核心方法/技巧：**
- 開啟一個 Agent 很便宜，但協調 20 個 Agent 的腦力成本會吃掉你所有省下的時間
- 「你無法在自己腦子裡成功管理 20 個 Agent」——Worktrees 解決的是機械碰撞，但你的 review 頻寬才是真正的天花板
- 解法：不是跑更少 Agent，而是設計好 Sub-agent 驗證機制，讓你「不在場時」也能信任輸出

**為什麼有用：**
很多人開始多開 Agent 卻發現效率反而下降——這個概念解釋了原因，也給出了方向。

**編輯觀點：** 💬 「Orchestration Tax 不還，技術債和注意力債同時累積」這句話是本週最值得貼在桌前的提醒。

**天馬行空的延伸：** 🚀 如果 Orchestration Tax 有「計算公式」：每加一個 Agent 要審查的 PR 增加 N 個，找出你個人的 N 值，就知道自己最多能跑幾個 Agent。

**💬 最有價值的回應：**
> "The orchestration tax left unpaid is how you accumulate both technical debt and attention debt at once."
— @addyosmani 原文（被 LinkedIn 廣泛引用），點出了雙重債務的本質。

**推文連結：** https://x.com/addyosmani/status/2059844244907696186

**類型：** 🚀 新工具

- 可同時啟動數百個並行 sub-agent 執行複雜任務（上限：單次最多 16 個並行，總計 1000 個）
- 配合 Opus 4.8 新預設模型同步推出；Enterprise/Team/Max 方案可用（Research Preview）
- 任務完成後 Claude 自行驗證結果

**為什麼有用：**
之前你需要自己寫多 Agent 協作腳本；現在你只要說「幫我做 X」，Claude 自己決定要召喚幾個 Agent、怎麼分工、怎麼驗收。

**天馬行空的延伸：** 🚀 試著讓 Dynamic Workflows 幫你做「一週 SEO 稽核」：一個 Agent 抓資料、一個分析弱點、一個生成修改建議、一個對比上週數據——全自動週報。

### 15. **gregisenberg：Sessions + Profiles 是 Agent 的「無聊魔法」**

**帳號：** @gregisenberg

**類型：** 💡 工作流優化

**核心方法/技巧：**
- Sessions（對話持久性）+ Profiles（個人偏好記憶）是讓 Agent 從「每次都像第一次見面」升級為「真正懂你的助理」的關鍵
- 缺少這兩者的 Agent 每次都從「金魚模式」重新啟動——耗時又耗 Token
- 實作策略：把你最常用的情境、偏好、工作流格式整理進 Profile；確保你用的 Agent 工具支援 Session 持久化

**為什麼有用：**
這是「多數人沒在認真做但差距很大」的基礎設施。建好個人 Profile 之後，每次 Agent 互動的品質會系統性提升。

**編輯觀點：** 💬 「無聊魔法」這個說法很準——Sessions + Profiles 本身不酷炫，但它決定了 Agent 能不能真正融入你的工作流，而不是每次都要重新教育它。

**天馬行空的延伸：** 🚀 建立你的「AI 個人檔案庫」：一個 YAML 或 Markdown 檔，記錄你的寫作風格、工作偏好、常用術語表、品牌語氣——每次啟動新 Agent 時作為 Profile 載入。

**💬 最有價值的回應：**
> "Once an agent can keep context, touch files/browser/tools, and not restart from goldfish mode every time — that's when agentic workflows actually become worth the setup cost."
— @gregisenberg，點出了 Agent 實用性的最低門檻。

**推文連結：** https://x.com/gregisenberg
